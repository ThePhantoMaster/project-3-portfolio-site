Static Sticky Relative Absolute Fixed

Static

This section has a default value of position: static;

position: static; is the default position value for HTML elements. It keeps content in the expected flow and any element that uses position: static; will ignore any applied top: [value];, right: [value];, bottom: [value];, left: [value];, or z-index: [value]; property/value pair.

Sticky

This section has a default value of position: static; however the examples within use position: sticky; coupled with a directional value like top: [value];.

position: sticky; checks the position of an object in relation to the viewport (screen). It keeps content in the expected flow until that object reaches the determined distance from one of the viewports edges as defined using one of the directional properties (top, right, bottom, left). Any element that uses position: sticky; also requires one of the directional properties defined as top: [value];, right: [value];, bottom: [value];, left: [value];, or z-index: [value]; property/value pair.

This object uses position: sticky; with right: 24px;.
This object uses position: sticky; with bottom: 24px;.
This object uses position: sticky; with left: 24px;.

Relative

This section has a default value of position: static; however the examples within use position: relative; coupled with a directional value like top: [value];.

position: relative; initially appears identical to position: static; until you pair it with a directional value. It keeps content in the expected flow but when paired with a directional vlaue it will adjust the position of that element the set distance of the directional value from its default placement. Meaning, top: 24px; will push the display of that element down (from the top) 24px;

Absolute

This section has a default value of position: static; however the example container uses position: relative; and the examples within use position: absolute; coupled with a directional value like top: [value];.

position: absolute; initially appears identical to position: fixed; until you pair it with a parent element that uses position: relative;. This pairing is the most common way that position: absolute; is used and my recommendation to you as well. It removes the content from the expected flow entirely, and will collapse its parent element if there is no other content within the parent element. When paired with a directional vlaue it will position the directional value of that element the set distance of the direction value from the same edge of its parent container. Meaning, top: 24px; will position the top edge of the element 24px from the top edge of the parent container.

This object uses position: absolute; with bottom: 24px;.
This object uses position: absolute; with left: 24px;.

Fixed

This section has a default value of position: static; however the examples within use position: fixed; coupled with a directional value like top: [value];.

position: fixed; is the most aggressive positioning mode as it always removes the content from the expected flow entirely, will collapse its parent element if there is no other content within the parent element, and will always position itself based on the viewport no matter what its parent element is set to. Meaning, top: 24px; will position the top edge of the element 24px from the top edge of the screen, always, even as you scroll.
